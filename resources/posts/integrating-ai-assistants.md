The landscape of software development has fundamentally shifted. Rather than writing every line of boilerplate code, my workflow is now heavily augmented by AI assistants. 

## Local LLMs and Privacy

While cloud-based models are powerful, I've started heavily relying on local LLMs running on my machine for tasks that involve sensitive client code. Using tools like Ollama or LM Studio, I can run models that are surprisingly capable of understanding my codebase context without sending data to a third-party server.

## My Automation Stack

1. **Code Generation:** Writing boilerplate controllers, tests, and database migrations.
2. **Refactoring:** Pointing an agent at a messy file and asking it to enforce clean code principles and early returns.
3. **Documentation:** Generating Markdown documentation based on my PHP docblocks and function signatures.

The trick is knowing exactly what AI is good at and what it isn't. I don't use AI to make architectural decisions, but I do use it to execute the mundane steps of implementing those decisions.
